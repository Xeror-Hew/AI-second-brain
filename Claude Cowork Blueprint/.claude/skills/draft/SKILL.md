# Draft

Produce a written deliverable the disciplined way. The whole trick is one rule.

## The core rule

```
SEPARATE PRODUCING FROM JUDGING
```

Drafting and editing are different jobs; doing both at once does neither well. Get a complete rough version down first, then make it good. Polishing sentence one before sentence two exists is how pages never get finished.

## The loop

1. **Outline.** Before writing, settle the one point and the shape: who it's for, what it should make them think or do, and the 3-7 beats that get there. Check the brief and run `/check-library` so you build on what exists and don't repeat it.
2. **Draft.** Write the whole thing rough, start to end, without stopping to polish. A complete weak draft beats half a perfect one. Drop a `[?]` where you're unsure rather than stalling.
3. **Read it cold.** Step back, then read it as the reader would, top to bottom. Mark what's unclear, unsupported, or off-point. Don't fix while reading; collect first.
4. **Revise.** Now fix: cut what doesn't serve the point, support every claim (a source where it matters), tighten the weak parts, resolve every `[?]`.
5. **Critique.** Run `/critique` for a fresh pair of eyes before you call it done, then apply the feedback with rigor.
6. **Finish.** Save it to the right file in the vault, then close out with `/done` (which updates the library).

## Self-review gate (before "done")

- Does it deliver the one point, for the intended reader?
- Is every claim supported? Sources where they matter?
- Did you cut the padding, or only add?
- Anything left ambiguous or unresolved?

No to any → back to revise. Verify with fresh eyes before claiming done.
