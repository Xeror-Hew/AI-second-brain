name: check-plan
description: Check the technical plan before changing architecture or a settled decision.
when_to_use: ALWAYS before you make an architecture/design call or change something already decided, ahead of implementing
---

Before deciding architecture or touching a settled decision:

1. Read `project_brain/plan/plan_tech.md` (decisions and architecture by phase), and `plan_why.md` if you need the reasoning.
2. Confirm the change lines up. If it contradicts a settled decision, raise that with the user before going ahead.
3. If the plan changed, update `project_brain/roadmap/roadmap.md` the same session.
