description: Check the library before producing or referencing work you don't already know.
when_to_use: ALWAYS before you produce something new or reference existing work you haven't seen this session, ahead of writing from scratch
---

Before you produce something new or reference work you don't know:

1. Open `project_brain/library/library_index.md` for the overview and the deliverable table.
2. Find the entry whose description matches what you need and read that piece.
3. Reuse what's there. Build on it, point to it, stay consistent with it. Don't duplicate or contradict it.

No library, or clearly stale? Run `/library` first. Knowing what already exists saves you from rewriting it.
