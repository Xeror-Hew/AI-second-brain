when_to_use: ALWAYS before you explore, navigate, or modify a part of the code you don't know yet this session, ahead of reading files loose
---

Before you explore or modify code you don't know:

1. Open `project_brain/code_map/map_index.md` for the top-level view and the fragment table.
2. Find the fragment whose description matches what you need and read it.
3. Then go to the code (use `path:line` when it helps you navigate).

No map, or clearly stale? Run `/map` first. The map gets you to the right part faster, then you read the code.
