Get oriented, read-only. This is the cheap way back in: rebuild context from a few high-signal docs instead of re-reading everything. Read in this order, stopping once you have enough:

1. `project_brain/next_step.md` for the active item.
2. `project_brain/roadmap/roadmap.md` for the current front and blockers.
3. For the direction: `project_brain/plan/plan_summary.md`.
4. For the lay of the workspace: `project_brain/work_map/work_map_index.md`.
5. For what already exists: `project_brain/library/library_index.md`.

Give the user a short, plain recap: where we stopped, the next step, blockers if any. Then, so they never face a blank page, remind them in everyday terms what they can ask for:

- "catch me up" → where we left off (this)
- "let's think through X" → shape it together (`/brainstorm`)
- "draft X" → produce a piece (`/draft`)
- "look into X" → research it (`/research`)
- "review this" → sharp feedback on a draft (`/critique`)
- "remember this" → save it for next time (`/remember`)

Stay read-only; this is orientation.
