# Research

Find something out properly: gather, verify, then synthesize. The discipline is one rule.

## The core rule

```
DON'T TRUST A SINGLE SOURCE FOR A LOAD-BEARING CLAIM
```

Anything the conclusion rests on gets a second, independent confirmation. One source is a lead, not a fact.

## The loop

1. **Frame the question.** Pin down exactly what you need to answer and how you'll know the answer is good enough. A vague question wastes the whole search.
2. **Gather, several angles.** Check what's already here first: run `/check-library` and look through the vault (dispatch the `brain-researcher` agent to keep the digging off the main thread). Then go outside: search the web and read the actual sources, not just the snippets. Pull from independent sources, not one site quoted five ways.
3. **Verify.** For each claim the answer leans on, confirm it against a second independent source. Separate fact from opinion, and note what you could not confirm. A claim you can't verify gets flagged, not dropped silently.
4. **Synthesize.** Answer the question directly. Cite each load-bearing claim's source. State your confidence and name the gaps, so the reader knows what's solid and what's thin.
5. **Save it.** Put the findings where they'll be found: a note in `notes/` or a deliverable in the vault, with the sources. `/remember` anything durable worth carrying across sessions, and update the library if it's a finished piece.

## Before "done"

- Is every load-bearing claim backed by a source, ideally two?
- Did you mark what's uncertain instead of papering over it?
- Does the synthesis actually answer the question you framed?
